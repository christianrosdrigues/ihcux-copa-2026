#  CopaNaMão – Protótipo de Baixa Fidelidade

##  Integrante

* CHRISTIAN DA SILVA RODRIGUES


##  Problema Focado

O principal problema escolhido foi a dificuldade e demora para comprar alimentos dentro do estádio, devido às longas filas nos quiosques.

Também foram considerados:

* Dificuldade de localização no estádio
* Perda de lances importantes durante deslocamentos


##  Solução Proposta

Foi desenvolvido o protótipo do aplicativo CopaNaMão, que permite ao torcedor:

* Pedir comida diretamente pelo celular
* Visualizar o mapa do estádio em tempo real
* Assistir replays de lances importantes
* Acompanhar o placar e estatísticas ao vivo



##  Justificativa de Design

O design foi pensado para um ambiente com:

* Muito barulho
* Distrações constantes
* Pouco tempo de atenção

Por isso, foram adotadas as seguintes decisões:

* Botões grandes e bem visíveis
* Pouco texto em cada tela
* Informações principais em destaque (placar, ações rápidas)
* Navegação simples e direta
* Alto contraste para facilitar leitura


##  Fluxo do Usuário

###  Pedido de Comida

1. Usuário acessa a tela inicial (Home)
2. Clica no botão Comida
3. Visualiza o cardápio
4. Adiciona itens ao carrinho
5. Acessa o carrinho
6. Finaliza o pagamento
7. Recebe notificação quando o pedido estiver pronto



###  Assistir Replay

1. Usuário acessa a tela inicial (Home)
2. Clica no botão Replay
3. Escolhe um lance recente
4. Assiste ao vídeo



###  Navegação no Estádio

1. Usuário acessa a tela inicial (Home)
2. Clica em Mapa
3. Visualiza sua localização atual
4. Seleciona um ponto de interesse (banheiro, portão, lanchonete)




##  Funcionalidades Principais

* Dashboard com placar ao vivo
* Mapa interativo do estádio
* Sistema de pedidos de comida
* Replays de lances em tempo real
* Perfil com ingresso digital



##  Considerações Finais

O protótipo foi desenvolvido com foco na usabilidade em ambientes extremos, garantindo rapidez, clareza e eficiência na interação do usuário durante o evento.
